# Deep Cuts unattended batch implementation plan

1. Preserve the single shared Deep Cuts discovery engine, immutable Aggits assets, opaque permanent URLs, analytics and automated email delivery.
2. Validate and normalize every CSV row before network research begins.
3. Resolve only direct destinations and prove artist identity with official cross-links or matching destination metadata.
4. Enforce the 98% evidence gate; reject and report instead of guessing.
5. Isolate artists, retry temporary network failures, cache checks, checkpoint every transition and support resume.
6. Reuse the existing edition creator and one Cloudflare deployment for all accepted artists.
7. Generate permanent machine-readable and plain-language reports.
8. Test the parser, gates, isolation and two-artist path before a full batch.
9. Publish accepted changes through GitHub; the existing deployment generates scan-tested QR assets, verifies live pages and sends delivery emails.

No batch code may change the locked Deep Cuts presentation or Aggits artwork.
